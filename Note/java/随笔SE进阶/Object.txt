-fineline();
	protected修饰
	JVM垃圾回收期自动调用
	执行时机
		java对象即将被垃圾回收器回收时调用
	SUN为程序员准备的垃圾销毁时机（如静态代码块是类加载时机）
	多制造点垃圾才能让JVM调用GC（如循环里创建对象）
	System.gc();建议启动垃圾回收器，（也可能不启动）
	这个方法已经过时了

-普通内部类
	可读性太差，不建议使用
-匿名内部类
	用接口名直接创建对象，在这里{}实现接口中方法
	new 接口名(){}
	用法
		接口名作方法参数，方法体中调用接口方法
		传递参数时直接用匿名内部类就可以了，会调用该类中的方法
	不建议使用，代码太乱，没有名字，不能复用，目的是读懂别人的代码
